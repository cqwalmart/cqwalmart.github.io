三、基础记分：raw_score
对每个学校 s，先算原始分：

raw_score[s] = Σ( question_weight[q] * option_vector[q][answer_q][s] )

说明：
- q 遍历 1..32 题
- answer_q 是用户在该题选择的 A/B/C/D
- option_vector 来自题库文件中该选项的“向量”
- question_weight 来自题库文件中该题的“建议题目权重”

四、当前推荐的主体算法（正常学校）
当前推荐主体算法：

mix_score[s] = 0.75 * z_score[s] + 0.25 * raw_centered[s]

只对正常学校 [0..11] 用这个主体算法排序。

五、z_score 的定义
z_score[s] = ( raw_score[s] - mean_random[s] ) / sd_random[s]

解释：
- mean_random[s]：在“每题四个选项等概率乱选”的条件下，学校 s 的随机期望分
- sd_random[s]：在同样条件下，学校 s 分数的随机标准差

为什么要有 z_score：
- 解决不同学校“入口池大小不一样”的问题
- 让结果比较的是“高于自己随机基线多少”，不是单纯比 raw 分高低

六、raw_centered 的定义
raw_centered[s] = ( raw_score[s] - mean_random[s] ) / max_pool[s]

解释：
- max_pool[s]：学校 s 在 32 题正式题库里，理论上能拿到的最大 raw 分
- raw_centered 的作用是：
 在保留“比随机基线高多少”的同时，再按学校自己的理论总池子压一遍
- 它比直接用 raw_score 更公平
- 它比 raw_norm = raw_score / max_pool 更保留“真实命中量”

七、为什么最终用 0.75 / 0.25
当前版本的思路是：

1）主体仍然让 z_score 决定
 - 因为 z_score 最能消掉入口不均的偏差
 - 更适合做后台校准

2）raw_centered 只做轻补偿
 - 目的是保留“这所学校确实在很多地方被真打中了”的手感
 - 不让结果被纯统计压得太平

当前固定系数：
- z_score 系数 = 0.75
- raw_centered 系数 = 0.25
